# Skill Roadmaps: Knowledge

Turn recurring skill gaps (JD trends × the owner's profile) into a concrete, sequenced
plan. The roadmap is DERIVED at runtime, this file holds the method, not a fixed plan
(a fixed plan belongs to one person and goes stale).

## How to build a roadmap
1. **Baseline** from PROFILE.md: strong tier (foundations to build on), basic tier
   (cheap upgrades), never-claim tier (true gaps).
2. **Demand** from 30 days of trends + atsKeywords: which gaps actually gate the
   owner's target roles in their market.
3. **Sequence by dependency and adjacency:**
   - Upgrades adjacent to strong-tier skills come first (fast, high credibility).
   - Foundational platform skills (e.g. a cloud provider for data roles, TypeScript
     for frontend roles) come before tools that sit on top of them.
   - Depth tools come after the foundation is real.
4. **Anchor every step to a project.** One end-to-end project on GitHub that exercises
   the top 2–3 gap skills beats five courses. Prefer extending a project already in
   base_cv.md, it converts to CV evidence immediately.
5. **Cap the plan.** 3–4 steps, one in flight at a time. Don't sprawl across 8 tools
   shallowly.

## Sequencing principle
[Foundation skill] + [orchestration/framework on top] + **one strong end-to-end
project** makes an owner competitive for their target band *fast*. Pair each new skill
with a real JD that demanded it so learning stays demand-driven (the LEARNING loop
surfaces these from actual postings).

## Per-market notes
Programs, bootcamps, and nationally-valued credentials specific to the owner's market
live in `job-search/knowledge/market_playbook.md`, cross-reference before
recommending.
