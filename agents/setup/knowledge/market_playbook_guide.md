# Market Playbook Guide

You generate `job-search/knowledge/market_playbook.md` for the owner's target market
using **live web research** during setup. This replaces any generic market assumptions, 
a job search in Berlin, Riyadh, São Paulo, or Toronto needs different platforms,
employers, and screening realities.

## Research (web search, ~10–15 minutes of effort)

1. **Platforms.** Which job boards actually carry the owner's target roles in their
   market? LinkedIn is near-universal; find the 1–3 regional boards that matter
   (e.g. StepStone/Xing in Germany, Bayt/GulfTalent in the Gulf, Indeed/Glassdoor in
   North America, Seek in Australia). Note each platform's strength and search URL shape.
2. **Employers.** 15–25 companies in that market that hire the owner's target roles at
   the owner's seniority. Mix: large anchors, well-funded scale-ups, respected local
   firms. Note hiring-program names (grad schemes, trainee tracks) where relevant.
3. **Salary bands.** Realistic ranges for the target roles at the owner's level, in
   local currency, with the source year. Mark estimates as estimates.
4. **Work-authorization reality.** How visa/nationality requirements show up in postings
   in this market (sponsorship language, citizen-only flags, quota schemes like
   Saudization/Emiratization, security clearances) and how the job-search agent should
   filter given the OWNER'S status from PROFILE.md.
5. **Market quirks.** CV norms (photo or not, 1 vs 2 pages, language), typical
   response times, screening-call culture, anything that changes agent behaviour.

## File shape (follow this skeleton)

```markdown
# [Market] Job Market Playbook, generated [date] for [owner's target roles]

## Platforms (ranked for this market)
| platform | strength | how to search |

## Target employers

## Salary reality ([currency], [year])
...

## Work authorization in this market

## CV & application norms here

## Refresh notes
Generated from web research on [date]. The manager re-validates quarterly;
job-search appends durable observations below.
```

## Example (fictional, for shape only)

> **Jane Doe, junior frontend, Berlin:** platforms LinkedIn + StepStone;
> employers: 20 named Berlin product companies and agencies; salary
> €42–55k junior; work auth: EU citizen → no filter needed, skip "German C2 required"
> roles (Jane is B1); CV norms: no photo needed in tech, 1 page fine for juniors.

Keep the file under ~150 lines, it's an operating manual, not a market report.
